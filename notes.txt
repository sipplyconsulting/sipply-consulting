GIT UPDATE WORKFLOW
--------------------
1. cd path/to/your/sipply-consulting-project
2. git status              (see what changed)
3. git add .                (stage all changes)
4. git commit -m "describe your change here"
5. git push                 (send to GitHub)

Notes:
- No need to repeat "git init" or "git remote add origin" — only done once when repo was first set up.
- "git push" alone works after the first push since upstream is already linked (-u origin main was set initially).
- Always write a clear commit message so you can track what changed later.

GIT STATUS SYMBOLS
--------------------
M   = Modified        (file changed, not yet staged)
A   = Added            (new file staged for commit)
D   = Deleted          (file removed)
R   = Renamed          (file path changed)
??  = Untracked        (new file Git doesn't know about yet)
U   = Unmerged         (conflict during merge, needs resolving)
C   = Copied           (rare, file copied and tracked as new)

Color meaning (in terminal):
Red    = changes not staged yet (need "git add")
Green  = changes staged, ready to commit ("git add" already run)

Example output of "git status":
  On branch main
  Changes not staged for commit:
    modified:   work.html          <-- red M
  Untracked files:
    IMG/work/project-alpha.jpg     <-- red ??

After running "git add .":
  Changes to be committed:
    modified:   work.html          <-- green M
    new file:   IMG/work/project-alpha.jpg   <-- green A


    GIT VIA VS CODE (NO TERMINAL)
------------------------------
1. Open Source Control panel (left sidebar icon)
2. Type commit message in top text box
3. Cmd+Enter (or click "Commit") to commit
4. Click cloud/sync icon at bottom to push
